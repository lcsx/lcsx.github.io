    # MD5

    # MD5加密的特点：

    1. 不可逆运算
    2. 对不同的数据加密的结果是定长的32位字符（不管文件多大都一样）
    3. 对相同的数据加密，得到的结果是一样的（也就是复制）。
    4. 抗修改性 : 信息“指纹”，对原数据进行任何改动,哪怕只修改一个字节,所得到的 MD5 值都有很大区别.
    5. 弱抗碰撞 : 已知原数据和其 MD5 值,想找到一个具有相同 MD5 值的数据(即伪造数据)是非常困难的.
    6. 强抗碰撞: 想找到两个不同数据,使他们具有相同的 MD5 值,是非常困难的

    MD5加密安全吗？其实是不安全的，不信的话可以到这个网站试试：md5破解网站。

    # 加“盐”

    加盐的目的是加强加密的复杂度，这么破解起来就更加麻烦 “盐”放在服务器，可以通过抓包泄露出去，还是不安全

    一般哈希算法是不可逆的，md5是，sha1，sha256，等都是。node内置有哈希算法的模块，可以直接用的

    # node 加密算法模块crypto

    const crypto = require('crypto');
//var Buffer = require("buffer").Buffer;  //解决中文不一致的BUG   如果不一致可以用这个


const getMd5 = data => {
    const md5 = crypto.createHash('md5');//创建一个md5 hash算法
    md5.update(data.toString())
    // md5.update(new Buffer(data).toString("binary"));//添加要转化的值
    return new Promise((resolve, reject) => {
        let passSolt = md5.digest('hex');//打印16进制的密文，
        resolve(passSolt)
    })
}


const getSha1 = data => {
    const sha1 = crypto.createHash('sha1');//创建一个sh1 hash算法
    sha1.update(data.toString());
    return new Promise((resolve, reject) => {
        const passSolt = sha1.digest('hex');//打印16进制的密文，
        resolve(passSolt)
    })
}


const getSha1WithSolt = data => {
    const hmac = crypto.createHmac('sha1', 'key');//创建一个带秘钥的sha1或者md5算法
    hmac.update(data.toString());
    return new Promise((resolve, reject) => {
        const passSolt = hmac.digest('hex');//打印16进制的密文，
        resolve(passSolt)
    })
}

module.exports = { getMd5, getSha1, getSha1WithSolt };
