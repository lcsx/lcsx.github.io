程序写的时候运行在3000端口，但服务器基于安全考虑，对外暴露的端口比较少，有80，有334，不改代码端口怎么正常运行呢？用nginx

# nginx

反向代理工具，把外网的80端口转到内网的3000端口

// 安装nginx
$ apt-get install nginx

//如何配置nginx 告诉你配置文件在哪里的
$ nginx -t
//返回一些信息 包括配置文库目录  /etc/nginx/nginx.conf

//编辑配置文件
vim /etc/nginx/nginx.cofg

//改什么好呢
//先注释掉默认的80端口配置
# include /etc/nginx/sites-enabled/*
//添加80端口转发
server{
	listen 80;
	server_name localhost;
	location / {
		proxy_pass http://127.0.0.1:3000
	}
}

//保存并退出
:wq
//只退出是q


//最好先检测下配置ok不ok,语法对不对
$ nginx -t

//如何让配置生效 
//1.重启nginx
$ [sudo] service nginx restart

//2.
2.service nginx reload


//搞定

# pm2管理进程

当我本地关掉gitbash 的时候，我x咧，服务器的node服务也停了~~~！ 如何使程序在后台不停运行呢-------pm2管理进程

我们要做的内容有

  1. 安装PM2
  2. 使用pm2启动、停止、重启、重载程序
  3. 使用pm2的日志、环境变量管理功能
//bash 登录到远程服务器后  cd 到zhihu* 

//安装
npm i pm2 -g

//运行
pm2 start app

//如何停止
//pm2 stop 名称

pm2 stop app
//关掉所有
pm2 stop all 

//如何知道名称
pm2 ls

//重启    这个重启是会终端服务的
pm2 restart app

//不间断重启
pm2 reload app

//添加环境变量
NODE_ENV=PRODUCTION pm2 start app --update-env

这个就是
"scripts": {
    "start": "cros-env NODE_ENV=production node app",
    "dev": "nodemon app"
},


//查看日志
pm2 log  app

//将日期时间，在日志中能显示，怎么办
NODE_ENV=PRODUCTION pm2 start app --update-env --log-date-formate "YYYY-MM-DD HH:mm"
