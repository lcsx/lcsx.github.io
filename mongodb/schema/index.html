# 用户数据schema

数据类别

  • 数字 Number
  • 字符串 String
  • 可枚举的类型 enum:[xx,xx] 可设置默认值
  • 数组字符串 {type:[{type:String}]}
  • 数组对象 { type: [ { 字段1:{type:Strgin,字段2:{type:String}}} ] }
const {Schema,model} = mongoose;
const userSchema = new Schema({
	__v:{type:Number,select:false},
	name:{type:String,required:true},
	password:{type:String,required:true,select:false},
	avatar_url:{type:String},  //头像
	gender:{type:String,enum:['male','female'],default:'male',required:true},  //性别 可枚举的类别，有默认值
	...
	locations:{type:}
	xx:{
		type:[
			{
				school:{type:String},
				major:{type:String},
				diploma:{type:Number,enum:[1,2,3,4,5]}
			}
		]
	}
})

# 字段过滤

正如你所想，在schema中增加【select:false】,这样就不会默认返回了

添加了select：false后 查询所有的用户，只想返回几个字段值；那如何再显示select：false的字段呢

  1. 写死的方法 mongoose model的写法 通过model.findById().select('+字段1[+字段2+...]')
class UsersCtl{
	...
	async findById(ctx){
		const user = await User.findById(ctx.params.id).select('+education')
	}
	...
}
  1. 动态写法，其实就是动态的修改第一方法的select的值

这个设计还挺巧妙的 get方法 xxx?fields=educations;bussniess;;lcs

class UsersCtl{
	...
	async findById(ctx){
		const { fields } = ctx.query;  //参数都再fields里，用;隔开   【educations;bussniess;;lcs】这部分的切分成数组
		//开始组装了
		const selectFields = fields.split(';').filter(f=>f).map(f=>' +'+f).join(''); //filter是去空值的
		const user = await User.findById(ctx.params.id).select(selectFields)
	
	}
	...
}

const selectFields = fields.split(';').filter(f=>f).map(f=>' +'+f).join('');

TIP

这里的...map(f=>${f})... 空格很重要，因为select是用+xx1 +xx2 形式的

TIP

Schema.Types.objectId 数据类型用于定义默认路径， 验证方式， 获取/设置方法，用于数据库查询的默认字段，以及其他针对字符串与数字的特性

这里 follwing是个数组，数组的值是mongodb的默认字段【_id】,关联到【User】schema

现在可以controller加

async listFollowing(ctx){
	const user = await Users.findById(ctx.params.id).select("+following");
	ctx.body = user
}

此时只是查询出某个id的following的数组列表值，也就是某用户的所有粉丝，如何去获取各个粉丝的数据，比如粉丝名字，头像等。 那需要对数组遍历么，嗯，第一个念头是这样，但mongoose提供了api可以直接操作了哈，如下

async listFollowing(ctx){
	
	//获取Users 的following select为true的字段
	const user = await Users.findById(ctx.params.id).select("+following").populate('following'); 
	ctx.body = user.following;
}

# 关注

流程 我是谁（从auth获取）---> 取自己的关注列表 ---> 判断是否重复关注 ----> push 所关注人的id到列表里 && save()

 async follow(ctx){
 	const me = await Users.findById(ctx.state.user._id).select('+following')
 	if(!me.following.map(item=>item.toString()).includes(ctx.params.id)){    
 		 if(ctx.state.user._id == ctx.params.id){
 			 ctx.throw(402,'不能自己关注自己')
 		 }else{
 			 console.log(me.following)
 			 me.following.push(ctx.params.id);
 			 me.save();  //mongoose的操作
 		 }
 	}else{
 		ctx.throw(402,'不能重复关注')
 	}
 	ctx.status = 204
 }

WARNING

me.following 是个mongoose的一个类型，不能直接用，需要转为字符串

虽然有这么一说，但我实际操作的时候，没有经过toString()也是可以正确执行的

# 取关

async unfollow(ctx){
	const me = await Users.findById(ctx.state.user._id).select('+following');
	console.log(me.following)
	const index = me.following.map(id=>id.toString()).indexOf(ctx.params.id);
	
	if(index > -1){
		me.following.splice(index,1);
		me.save()
	}
	ctx.status = 204;
}

async checkUserExist(ctx,next){

	await next();
}
