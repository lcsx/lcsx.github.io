# 中间件 & 洋葱模型

在koa里，若是想执行下一个中间件，需next

洋葱模型 执行顺序

app.use(async(ctx,next)=>{
	console.log(1)
	await next();
	console.log(2)
	ctx.body = 'hello world'
})

app.use(async(ctx,next)=>{
	console.log(3)
	await next();
	console.log(4)
})
app.use(async(ctx)=>{
	console.log(5)
})

// 执行顺序  1 3 5 4 2

在koa中，是一个中间件

  1. 处理不同是url

  2. 处理不同HTTP方法

  3. 解析URL上的参数

  4. 获取url ctx.url

  5. 获取方法 ctx.method

  6. 获取参数

const userId = ctx.url.match(/\/users\/(\w+)/)[1]; //添加了（）使之分割为数组了

更优雅的实现：使用koa-router npm i koa-router --save

const Koa = require('koa');
const Router = require('koa-router');
const app = new Koa();
const router = new Router();

router.get('/',(ctx)=>{
	ctx.body = 'xxx'
});

router.get('/users',(ctx)=>{
	ctx.body = 'xx'
});

router.post('/user',(ctx)=>{    //get /post
	ctx.body = 'xxx'
})

router.get('/users/:id',(ctx)=>{   //不用正则匹配，方便了很多
	ctx.body = `${ctx.params.id}`
})
app.use(router.routes()); //注册中路由间件

# options

options作用

  1. 检测服务器所支持的请求方法 返回体里，Headers-->Allow里有支持的请求类型
  2. CORS中的预检请求

# allowedMethods的作用

  1. 响应options方法，告诉他所支持的请求方法 app.use(usersRouter.allowedMethods()) 这样接口就支持options请求了,即请求方法为options时，不会报错
  2. 响应的返回405（不允许）和501（没实现）

# 控制器

路由是拿到不同url分配不同的任务，控制器是拿到路由分配的任务去执行

  1. 拿到路由分配的任务，并执行
  2. 在koa中，是中间件

控制器作用

  1. 获取HTTP请求参数
  2. 处理业务逻辑（计算数据，存储数据）
  3. 发送http响应

# 获取HTTP请求参数

  1. Query String，如?q = keyword (路由参数的可选的)
  2. Router Params，如/users/:id (路由参数必须) 这里id参数为字符了，需要转为数字型，可以 ctx.params.id * 1 转换程数字类型
  3. Body，如{name:'xx'}
  4. Header,accpet/cookie

1.2 长度浏览器限制，所有需要3.body

koa是不支持解析请求体的，需要安装中间件 npm i koa-bodyparser --save

const bodyparser = require('koa-bodyparser');  //引入

# 发送HTTP响应

  1. 发送Status 200/400
ctx.status = 204
  1. 发送body
ctx.boy='<h1>ss</h1>'
  1. 发送Header，如allow/content-type
ctx.set('Allow','GET,POST');

编写控制器实践

  1. 每个资源的控制器放在不同的文件里
  2. 尽量使用类+类方法形式编写控制器
  3. 严谨的错误处理

const router = require('./routes/home.js');

app.use(router.routes()); //注册中路由间件

当有很多个路由时，要堆一堆app.use()来注册路由中间件么，可以优化，自动注册 /routes/index.js

const fs = require('fs');

const fn = (app) => {
    fs.readdirSync(__dirname).forEach(file=>{
        if(file === 'index.js') {return ;}
        const route = require(`./${file}`);
        app.use(route.routes()).use(route.allowedMethods());
    })
}

module.exports = fn;
