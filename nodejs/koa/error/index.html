# 错误处理

koa自带错误处理，如没有某接口，自动报出404错误 通过ctx.thorw(错误码,'错误信息')可以定义

# 错误处理中间件

app.use(async(ctx,next)=>{
	try{
		await next();
	}catch(err){
		ctx.status = err.status || err.statusCode || 500;
		ctx.body = {
			message:err.message
		}
	}
})

控制器里手动抛出的错误可以在上面捕获到throw()

class xx{
	index(ctx){
		...
		ctx.throw(412,'xxx')  //但无法处理404错误
	}
	...
}

# koa-json-error

错误处理，可以处理404

npm i koa-json-error --save

使用

const error = require('koa-json-error');


//不用以上方式，直接
app.use(error())

报错信息里有stack，堆栈信息，再生产环境不应该爆出来,需要配置下，

app.use(error({
	postFormat:(e,{stack,...rest})=process.env.NODE_ENV === 'production' ? rest:{stack,...rest}
}));

//营造生产环境，可以写在script
//非windows中可以直接
{
	"scripts":{
		"start":"NODE_ENV=production nodemon app"
	}
}

windows 可以安装一个跨平台设置环境变量的插件 cnpm i cross-env --save-dev (开发阶段用)

{
	...
	"scripts":{
		"start":"cros-env NODE_ENV=production node app",
		"dev":"nodemon app"
	}
}
